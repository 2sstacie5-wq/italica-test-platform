# Тест A1 з голосовими відповідями — платформа для мовної школи

Готовий сайт для тестування рівня учнів: письмові питання (Частини 1–7) з автоматичною
перевіркою + фінальна частина (усна й письмова продукція), де учень записує голосові
відповіді прямо в браузері. Усі відповіді, тексти й аудіофайли зберігаються на сервері,
і є окрема захищена паролем панель для вчителя — переглянути, прослухати аудіо та
виставити фінальну оцінку.

## Що є в комплекті

- `server.js`, `grading.js`, `store.js`, `test-data.js` — бекенд на Node.js/Express.
  `test-data.js` містить увесь тест (питання + правильні відповіді) — саме тут
  редагувати або додавати нові тести в майбутньому.
- `public/` — сторінка, яку бачить учень (проходження тесту, таймер на кожну частину,
  запис голосу через мікрофон).
- `admin/` — панель вчителя (захищена логіном/паролем): список здач, прослуховування
  аудіо, ручна перевірка Частини 7 та фінальної продукції, підсумковий бал.
- `data/` — тут з'являться `submissions.json` (усі відповіді) та `audio/` (аудіофайли).
  Створюється автоматично при першому запуску.

## Як влаштована перевірка

- Частини 1–6 (90 балів) перевіряються автоматично одразу після відправки тесту.
  Порівняння відповідей нечутливе до регістру, розділових знаків і діакритичних
  знаків (é/è тощо) — щоб не карати учня за клавіатуру.
- Частина 7 (10 балів) і фінальна усна/письмова продукція завжди перевіряються
  вручну вчителем у панелі admin — це відповіді у вільній формі, автоматично
  оцінити їх неможливо (та й не варто).
- У панелі admin вчитель може також **вручну виправити будь-яку автоматичну оцінку**
  (наприклад, якщо автоперевірка занадто строга до синоніма) — поле з балами
  редагується прямо в інтерфейсі.
- Прохідний бал — 60/100 (це задано в `test-data.js` → `meta.passPercent`,
  можна змінити).

## Запуск локально (для тестування)

Потрібен встановлений [Node.js](https://nodejs.org) (версія 18 або новіша).

```bash
cd italian-test-platform
npm install
cp .env.example .env      # відкрий .env і зміни ADMIN_PASSWORD
npm start
```

Сайт для учнів: http://localhost:3000
Панель вчителя: http://localhost:3000/admin (логін/пароль з .env)

Локально мікрофон працює без HTTPS (localhost — виняток у браузерах).

## Розгортання в інтернеті (щоб учні заходили з будь-якого пристрою)

Важливо: запис голосу (`getUserMedia`) браузери дозволяють **тільки на HTTPS**
(або на localhost). Тож потрібен хостинг з HTTPS "з коробки".

Також важливо: усі відповіді та аудіофайли зберігаються як звичайні файли на диску
сервера (`data/`). Це означає, що хостинг має бути з **постійним диском**, інакше
дані зникнуть після перезапуску/оновлення сервера. Варіанти:

1. **Render.com** — найпростіше. Створи "Web Service" з цього коду, вкажи
   `npm install` як build command і `npm start` як start command, додай змінні
   середовища `ADMIN_USER`/`ADMIN_PASSWORD`. На безкоштовному плані диск не
   зберігається між релізами — для реального використання потрібен платний план
   (від ~$7/міс) з підключеним Persistent Disk (примонтувати до папки `data/`).
2. **Railway.app** — аналогічно Render, є платні persistent volumes.
3. **Власний VPS** (DigitalOcean, Hetzner тощо, від ~$5/міс) — найнадійніше і
   найдешевше для довготривалого використання: встановлюєш Node.js, запускаєш
   через `pm2` або `systemd`, ставиш Nginx + безкоштовний сертифікат Let's
   Encrypt для HTTPS. Дані живуть на диску сервера скільки завгодно.

Якщо потрібна допомога з розгортанням на конкретному хостингу — скажи, і я підготую
покрокову інструкцію саме під нього.

## Резервне копіювання

Періодично копіюй файл `data/submissions.json` і папку `data/audio/` кудись окремо
(Google Drive, зовнішній диск) — це весь архів здач учнів.

## Як додати новий тест (B1, B2 тощо) у майбутньому

Уся логіка (таймери, типи питань, перевірка, панель вчителя) написана узагальнено.
Щоб додати новий тест, потрібно продублювати структуру `test-data.js` під новий
рівень (можна звернутись, і я підготую окремий тест і/або перемикач між тестами
на стартовій сторінці).

## Безпека

- Обов'язково зміни `ADMIN_PASSWORD` у `.env` перед тим, як викладати сайт в
  інтернет — за замовчуванням стоїть слабкий пароль.
- Панель `/admin` захищена HTTP Basic Auth — цього достатньо для невеликої школи,
  але не показуй прямий лінк стороннім людям.
